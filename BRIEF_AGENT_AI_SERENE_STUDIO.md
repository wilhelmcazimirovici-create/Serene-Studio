# Brief standalone pentru agent AI: website nou Serene Studio

Data: 30 aprilie 2026  
Destinatar: agent AI care porneste de la zero si nu are acces la proiectul Popcorn Hostel  
Obiectiv principal: creeaza un website nou pentru `serenestudio.ro` care genereaza cat mai multe rezervari online prin Beds24 si reduce la minimum contactarea personalului.

## 1. Misiunea

Construieste de la zero un website rapid, mobile-first, optimizat SEO/AEO/GEO pentru Serene Studio Bucuresti, un studio privat de inchiriat in regim hotelier/Airbnb, aflat langa Piata Muncii, Arena Nationala si mai multe spitale importante din estul Bucurestiului.

Site-ul trebuie sa faca trei lucruri foarte bine:

1. Sa convinga vizitatorul ca Serene Studio este cazarea potrivita.
2. Sa il trimita rapid catre rezervarea online Beds24, fara apeluri telefonice.
3. Sa raspunda pe site la toate intrebarile normale inainte si dupa rezervare.

Nu construi un site generic de hotel. Nu construi o pagina de prezentare frumoasa dar slaba comercial. Serene Studio este un singur studio privat, linistit, cu self check-in, chicineta, curte/terasa, Wi-Fi rapid si rezervare online.

## 2. Principiul central de conversie

Toata arhitectura trebuie sa reduca frictiunea pana la rezervare.

Reguli:

- Butonul `Rezerva acum` trebuie sa fie vizibil in primul ecran pe mobil.
- Pe mobil trebuie sa existe o bara sticky discreta jos cu `Rezerva acum`.
- Nu pune telefonul ca actiune principala. Telefonul este doar pentru urgente reale sau clarificari dupa rezervare.
- Preturile si disponibilitatea se verifica doar prin Beds24.
- Fiecare pagina importanta trebuie sa aiba CTA catre Beds24 in partea de sus, la mijloc si la final.
- Nu trimite vizitatorul spre Booking.com/Airbnb/Expedia daca scopul este rezervarea directa.
- Nu afisa linkuri sociale sau externe mai vizibile decat rezervarea.
- Nu cere utilizatorului sa trimita formular pentru disponibilitate. Disponibilitatea se verifica online.
- FAQ-ul trebuie sa raspunda la intrebarile care altfel ar genera apeluri sau mesaje.

Formularile recomandate:

- `Rezerva online - confirmare rapida`
- `Verifica disponibilitatea`
- `Vezi pretul pentru datele tale`
- `Rezervarile se fac online, prin sistem securizat Beds24`
- `Pentru preturi si disponibilitate, apasa Rezerva acum`

Formulari de evitat:

- `Contacteaza-ne pentru rezervare`
- `Suna pentru disponibilitate`
- `Trimite-ne un mesaj`
- `Pret la cerere`

## 5. Beds24: ce trebuie citit si cum se integreaza

Inainte sa implementezi, citeste documentatia oficiala Beds24:

- Booking Widgets: `https://beds24.com/booking-widgets.html`
- WordPress plugin: `https://wordpress.beds24.com/wordpress-plugin/`
- WordPress.org plugin: `https://wordpress.org/plugins/beds24-online-booking/`
- Embedded iframe: `https://wiki.beds24.com/index.php/Embedded_Iframe`
- Booking Button: `https://wiki.beds24.com/index.php/Booking_Button`
- Tracking bookings / referer: `https://wiki.beds24.com/index.php/Track_Bookings`

Rezumat operational din documentatia Beds24:

- Beds24 ofera booking widgets: availability calendar, booking box, booking strip, booking button si iframe.
- Widgeturile se genereaza in Beds24 din `Settings > Booking Engine > Booking Widgets`.
- Widgeturile pot fi integrate in orice site prin cod snippet.
- Pentru WordPress exista plugin oficial gratuit, bazat pe shortcodes.
- Pluginul cere un Beds24 ID si apoi permite adaugarea widgeturilor in pagini/postari/sidebar.
- Beds24 permite folosirea parametrului `referer` pentru a vedea din ce link/widget a venit rezervarea.
- Iframe-ul este posibil, dar Beds24 avertizeaza ca poate avea UX suboptim; pe mobil iframe-urile sunt problematice si trebuie testate atent.
- Booking page-ul Beds24 este responsive si poate fi customizat in Beds24.

### Recomandarea de integrare

Nu face iframe-ul solutia principala pe mobil. Solutia principala trebuie sa fie:

1. CTA nativ pe site: `Rezerva acum` / `Verifica disponibilitatea`.
2. Link direct catre Beds24:
   `https://beds24.com/booking.php?propid=249069&referer=website`
3. Optional: booking strip sau booking box Beds24 in home si pe pagina `/rezerva/`, daca widgetul generat este rapid, responsive si nu strica layoutul.
4. Optional: iframe doar pe pagina dedicata `/rezerva/`, doar daca testele mobile arata ca este comod. Pe mobil, ofera fallback clar: `Deschide pagina securizata Beds24`.

### Daca site-ul este WordPress

Daca decizi sa pastrezi WordPress:

- Instaleaza pluginul oficial `Beds24 Online Booking`.
- Configureaza Beds24 ID-ul corect.
- Genereaza shortcodes din Beds24 control panel.
- Creeaza o pagina `/rezerva/` cu shortcode pentru booking widget sau embed.
- Adauga in header, hero, sticky mobile bar si footer butoane care duc la booking.
- Testeaza pluginul pe mobil, inclusiv selectarea datelor, numarul de nopti si finalizarea rezervarii.

### Daca site-ul este static/custom

Daca faci site nou in Astro/Next/Nuxt/static HTML:

- Nu folosi plugin WordPress.
- Foloseste direct codul snippet generat de Beds24 pentru booking box/strip.
- Pastreaza mereu linkul direct Beds24 ca fallback.
- Pentru fiecare CTA foloseste un `referer` diferit, ca sa se poata masura conversiile:
  - `referer=website_home_hero`
  - `referer=website_mobile_sticky`
  - `referer=website_arena_page`
  - `referer=website_hospital_page`
  - `referer=website_faq`
- Daca Beds24 nu accepta spatii/caractere speciale in referer, foloseste doar litere mici, cifre si underscore.

## 6. Stack recomandat

Varianta preferata: site static rapid, fara WordPress, construit cu Astro sau Next static export.

Motive:

- viteza mai buna
- suprafata de atac mai mica
- control mai bun pe schema, hreflang, sitemap, llms.txt si performanta mobile
- mai putine pluginuri inutile

Varianta acceptata: WordPress curatat si optimizat, doar daca proprietarul vrea administrare prin WP.

Indiferent de stack:

- HTML semantic.
- CSS mobile-first.
- JS minim.
- Imagini optimizate WebP/AVIF.
- Date structurate JSON-LD.
- Sitemap XML.
- Robots.txt.
- llms.txt.
- Cookie banner daca folosesti analytics/Clarity/Maps embeds.
- Fara template hotel generic.

## 21. Mobile-first: criterii stricte

Testul principal este pe `360x640`.

Criterii:

- Hero-ul incape bine pe primul ecran.
- CTA-ul principal este vizibil fara scroll lung.
- Bara sticky de rezervare nu acopera continut important.
- Toate butoanele au minimum 44px inaltime.
- Meniul mobil este clar si rapid.
- Widgetul Beds24 nu depaseste latimea ecranului.
- Iframe-ul Beds24, daca exista, nu produce scroll dublu confuz.
- Textul nu se suprapune peste imagini.
- Cardurile nu au inaltimi instabile.
- Fonturile sunt lizibile: body minim 16px.
- Imaginile au dimensiuni setate si nu produc layout shift.
- Pagina se incarca rapid pe conexiune mobila.

Test viewports:

- 360x640
- 390x844
- 768x1024
- 1366x768
- 1440x900
